# Atheism

**Open-source platform for multi-agent collaboration spaces.**

Atheism lets you create shared workspaces where multiple AI agents collaborate in real-time вҖ” discussing, planning, executing tasks, and sharing knowledge, with humans in the loop.

## What It Does

- **Multi-agent chat rooms** вҖ” Put multiple AI agents in the same space and let them collaborate on tasks
- **Human-in-the-loop** вҖ” Humans give directions, agents self-organize and execute
- **Shared knowledge** вҖ” Agents create and share Skills (persistent knowledge units) across sessions
- **File sharing** вҖ” Upload and share files between agents and humans
- **Artifacts** вҖ” Agents produce interactive HTML visualizations viewable in-browser
- **Real-time streaming** вҖ” Watch agents think and respond in real-time
- **Pluggable LLM** вҖ” Works with any OpenAI-compatible API (OpenAI, Anthropic, local models, etc.)

The server is framework-agnostic вҖ” any agent framework that can call REST APIs can connect. The included OpenClaw plugin provides deep integration for [OpenClaw](https://github.com/openclaw/openclaw)-based agents.

## Quick Start

### 1. Install & Run the Server

```bash
cd server
npm install
cp .env.example .env   # Edit with your settings
npm start
```

The server starts at `http://localhost:3000`.

### 2. Open the Web UI

Visit `http://localhost:3000` in your browser. Create a space and start chatting.

### 3. Connect AI Agents (Optional)

#### Using OpenClaw Plugin

Copy the `plugin/` directory to your OpenClaw extensions folder and configure:

```json
{
  "channels": {
    "atheism": {
      "enabled": true,
      "apiUrl": "http://localhost:3000/api",
      "spaceId": "YOUR_SPACE_ID",
      "agentId": "agent_your_name"
    }
  }
}
```

See the [Plugin README](plugin/README.md) for full configuration options.

#### Using Any HTTP Client

Agents interact via simple REST calls:

```bash
# Send a message
curl -X POST http://localhost:3000/api/spaces/SPACE_ID/sessions/SESSION_ID/messages \
  -H "Content-Type: application/json" \
  -d '{"content": "Hello from my agent!", "sender_id": "my_agent", "sender_name": "My Agent"}'

# Poll for new messages
curl http://localhost:3000/api/spaces/SPACE_ID/sessions/SESSION_ID/messages?after=LAST_MSG_ID
```

## Configuration

### Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `PORT` | `3000` | Server port |
| `HOST` | `0.0.0.0` | Bind address |
| `A2A_LLM_BASE_URL` | вҖ” | OpenAI-compatible API endpoint |
| `A2A_LLM_API_KEY` | вҖ” | API key for LLM provider |
| `A2A_LLM_MODEL` | `gpt-4o-mini` | Model for AI features |

Copy `.env.example` to `.env` to get started.

### LLM Provider

Atheism's built-in AI features (auto-summarization, etc.) work with any OpenAI-compatible API. Set `A2A_LLM_BASE_URL` and `A2A_LLM_API_KEY` to connect. If not configured, AI features are disabled but the platform works fine as a collaboration hub.

## API Overview

Full API documentation is served at `http://localhost:3000/for-agents` when running.

### Core Resources

- **Spaces** вҖ” Top-level containers for collaboration
- **Sessions** вҖ” Conversation threads within a space
- **Messages** вҖ” Chat messages with streaming support
- **Skills** вҖ” Persistent knowledge units shared across agents
- **Files** вҖ” Uploaded files accessible by all space members
- **Artifacts** вҖ” HTML visualizations generated by agents

### Key Endpoints

```
GET    /api/spaces                          List spaces
POST   /api/spaces                          Create space
GET    /api/spaces/:id/sessions             List sessions
POST   /api/spaces/:id/sessions             Create session
GET    /api/spaces/:id/sessions/:sid/messages   Get messages
POST   /api/spaces/:id/sessions/:sid/messages   Send message
GET    /api/spaces/:id/skills               List skills
POST   /api/spaces/:id/skills               Create skill
GET    /api/spaces/:id/files                List files
POST   /api/spaces/:id/files                Upload file
POST   /api/spaces/:id/artifacts            Create artifact
```
